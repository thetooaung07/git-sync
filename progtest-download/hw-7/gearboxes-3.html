The task is to develop a program that designs gear boxes.

Assume there exist prefabricated gearboxes. A gearbox is defined by the gear ratio - the number of teeth of its cogwheels. The number of teeth must be between 10 and 1000. A gearbox with gear ratio e.g., 13:17 may be used to increase the speed (13:17) or reversed to decrease the speed (17:13).

The program is given a list of gearboxes and a list of gear ratios. The task is to find some combination of the gearboxes to produce the desired gear ratio. For example, the program is given the following gearboxes:

+ 13:17
+ 15:12
+ 11:24
+ 18:16
+ 27:47

The task is to combine (some) of the gearboxes and produce gear ratio 30:11. We may combine gearboxes 15:12 and 24:11 (reversed gearbox 11:24). The resulting ratio exactly matches 30:11. There exist inputs where the ratio cannot be matched exactly (e.g., ratio 75:14). In this case, the program finds some combination of the gearboxes such that the combined gear ratio is as close as possible. See section Advice for the discussion on how to compare the ratios.

First, the input lists the existing gearboxes. A gearbox is entered as + X:Y, where X and Y defines the number of teeth of the cogwheels. Following the gearboxes, there is a list of problems to solve. A problem is entered as ? X:Y where X:Y is the desired gear ratio. The input of gearboxes and problems is not interleaved, i.e, no further gearboxes are entered after the first problem in the input.

The output of the program is the solution of each input problem. The solution prints out the computed gear ratio that is the closest to the desired ratio. Following the computed ratio, there is a list of gearboxes that are used to produce that ratio. There may exist more than one correct answer, any correct answer is accepted by the testing environment. Sometimes, the best solution is gear ratio 1:1. The list of gearboxes is empty in such case (see the example run).

The program must validate input data. If the input is invalid, the program must detect it, it shall output an error message (see below) and terminate. If displayed, the error message must be displayed on the standard output (do not send it to the error output) and the error message must be terminated by a newline (\n). The input is considered invalid, if:

  • an invalid gearbox in the input (missing a separator or a number),
  • the number of teeth of a cogwheel is outside of the limits (a cogwheel must have between 10 and 1000 teeth),
  • an invalid format of the problem (missing a separator or a number),
  • an invalid gear ration in the problem (negative or zero value). There are not any further restrictions on the numbers in the desired ratios, e.g., a gear ratio 1:5000 is perfectly valid,
  • the gearboxes and problems are interleaved in the input (i.e., there is a new gearbox entered after a problem).

Your program will be tested in a restricted environment. The testing environment limits running time and available memory. The expected solution checks all possible gear box combinations and selects the combination closest to the desired gear ratio. This may need to test an enormous number of cases, thus the implementation must be reasonably efficient.

Advice:
  • The example runs above list both the output of your program (boldface font) and user input (regular font). The bold/regular formatting is included here, in the problem statement page, to increase readability of the listing. Your program must output the text without any additional markup.
  • Do not forget the newline (\n) after the last output line.
  • There is at most 50 gearboxes in the input.
  • Use long long (64 bit) integers to represent the gear ratios. Smaller data type (32 bit int) is likely to overflow. A special care must be taken when choosing the resulting gear ratio (the ratio closest to the desired ratio). We recommend to use floating point values (double type) for the comparison. The comparison is a bit tricky. Consider the example with gearbox 500:10 and desired gear ratio 1:3 (=0.3333...). We have two options: either use the gearbox (reversed, i.e., gear ratio is 10:500 = 0.02) or not use any gearbox (gear ratio is 1:1 = 1.0). A comparison of the floating point values chooses gearbox 10:500 (of the two options, 0.02 is the one closer to 0.3333...). But the gearbox 10:500 decreases the speed by a factor of 50 (thus the speed is 16.666... times slower than desired) while no gearbox option leaves the speed only 3 times faster than desired. Thus the comparison needs to be done in a more complicated way:
    1. divide the newly found gear ratio a with the desired gear ratio x, i.e., compute the fraction a / x. If the result is smaller than 1, replace the fraction by its reciprocal (x / a). The ideal fraction would be 1, the higher the fraction is, the more the gear ratios differ. Note that the fraction cannot be below 1,
    2. divide the so-far best gear ratio b with the desired gear ratio x, i.e., compute the fraction b / x. Again, if the ratio is less than 1, use the reciprocal fraction x / b instead,
    3. if the fraction from step (1) is smaller than the fraction from step (2), then the new gear ratio a is closer to the desired result than so-far best gear ratio b.
