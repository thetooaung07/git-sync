The task is to develop a program to compute the size of a box to pack some products.

We assume that our factory produces several different products. The product are always of a prismatic shape, however, their size varies. The product shall be packed into boxes. There are some requirements on the box: the box must be a cube, the products must occupy the whole volume of the box (no extra space), and the box must be universal (usable for any of our products). We always pack products of one kind into the box. We do not have any restriction on the number of products in the box. The problem is to find the size of that universal box.

The input of the program are parameters of our products. The input is formed by three-tuples of decimal numbers, each three-tuple describes the size of some product. The decimal numbers always use two decimal places and are in the range from 0.01 to 10000000.00 inclusive. The total number of products is not explicitly limited. The input ends when there is active EOF (end-of-file) in the standard input.

The output of the program is the computed size of the box. The size is shown using two decimal places in the format shown below. If the size of the box exceeds 10000000.00, the program prints out warning about the size (see below). Anyway, the answer is printed with a newline character (\n) at the end.

The program must validate input data. If the input is invalid, the program must detect it, it shall output an error message (see below) and terminate. If displayed, the error message must be displayed on the standard output (do not send it to the error output) and the error message must be terminated by a newline (\n). The input is considered invalid, if:

  • there are no data in the input (no products),
  • there were not three numbers for some product,
  • some size was not a decimal number,
  • some size was not in the range from 0.01 to 10000000.00 inclusive.

Further, the program shall check whether the input decimal numbers were specified with two decimal places. This validation requires some extra programming, thus it is not required in the mandatory tests (the mandatory tests always enter decimals with two decimal places). There is a bonus test to check this extra validation. The bonus test inputs decimals with less or more decimal places that shall be rejected by the program. If you extend your program to pass this extra test, you earn some extra point bonus.

Please strictly adhere to the format of the output. The output must exactly match the output of the reference program. The comparison is done by a machine, the machine requires an exact match. If your program provides output different from the reference, the program is considered malfunctioning. Be very careful, the machine is sensitive event to whitespace characters (spaces, newlines, tabulators). Please note that all output lines are followed by a newline character (\n). This applies even to the last line of the output, moreover, this applies even to the error message. Download the enclosed archive. The archive contains a set of testing inputs and the expected outputs. Read Progtest FAQ to learn how to use input/output redirection and how to simplify testing of your programs.

Your program will be tested in a restricted environment. The testing environment limits running time and available memory. The exact time and memory limits are shown in the reference solution testing log. However, neither time nor memory limit could cause a problem in this simple program.
